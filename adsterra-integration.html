Adsterra Integration Guide

Complete guide for integrating Adsterra ads with privacy compliance, performance optimization, and user experience best practices.

üöÄ Quick Start

Note: This integration includes privacy controls, consent management, and performance optimization to ensure the best user experience while maximizing revenue.

  1. Get Your Adsterra Account

    Sign up for an Adsterra account and get your Site ID and placement IDs from the dashboard.

  2. Update Configuration

    Replace the placeholder values in ad-manager.js with your actual Adsterra credentials:

    // Update this in ad-manager.js siteId: 'YOUR_ACTUAL_ADSTERRA_SITE_ID', placements: { header: { id: 'YOUR_HEADER_PLACEMENT_ID', type: 'banner', size: '728x90' }, // ... other placements }

  3. Include the Scripts

    Add the following scripts to your HTML before the closing body tag:

    <!-- Ad Manager --> <link rel="stylesheet" href="ad-styles.css"> <script src="ad-manager.js"></script>

  4. Test Integration

    Load your website and check the browser console for "Adsterra script loaded successfully" message.

‚öôÔ∏è Configuration Options

Ad Placements

The system supports multiple ad placements with automatic responsive sizing:

// Available placements placements: { header: { size: '728x90' }, // Top banner sidebar: { size: '300x250' }, // Side banner footer: { size: '728x90' }, // Bottom banner inContent: { size: '300x250' }, // In-content banner popunder: { type: 'popunder' } // Pop-under ads }

Privacy Settings

privacy: { requireConsent: true, // Require user consent respectDNT: true, // Respect Do Not Track gdprCompliant: true // GDPR compliance }

Performance Settings

performance: { lazyLoading: true, // Enable lazy loading preloadCritical: false, // Preload critical ads maxRetries: 3 // Maximum retry attempts }

üé® Customization

Ad Container Styling

Customize ad appearance by modifying the CSS classes in ad-styles.css:

/* Custom ad container styles */ .ad-container { margin: 2rem 0; border-radius: 12px; overflow: hidden; } .ad-placeholder { background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border: none; }

Consent Dialog Customization

Modify the consent dialog appearance and messaging:

/* Custom consent dialog */ .ad-consent-dialog .consent-content { background: your-custom-bg; border-radius: 16px; box-shadow: 0 25px 50px rgba(0, 0, 0, 0.15); }

üì± Responsive Design

Mobile-Optimized: All ad placements automatically adjust for mobile devices with appropriate sizing and positioning.

The ad system includes responsive breakpoints:

  • Desktop: 728x90, 300x250, 300x600, 160x600
  • Tablet: 468x60, 300x250
  • Mobile: 320x50, 300x250

Best Practices

  • Test ad placements on different devices and screen sizes
  • Monitor Core Web Vitals to ensure performance isn't degraded
  • Respect user privacy preferences and provide easy opt-out options
  • Regularly update ad configurations based on performance data
  • Consider user experience when placing ads (avoid intrusive placements)
